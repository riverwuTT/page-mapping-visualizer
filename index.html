Interactive JavaScript ports of tt-metal's data-layout model — how a tensor's elements end up scattered across device cores. Two layers, two pages:

The full story: a logical N-D tensor shape, a layout (row-major or 32×32 tile), and a sharding scheme map every element to a page, then to a core. Ported from tensor_spec.hpp.

How they relate

The buffer model assumes shapes are already in pages. The tensor model adds the missing step in front of it: folding an element shape into pages via the layout (compute_physical_shape → get_page_shape), then handing the resulting page grid + shard shape to the very same page → core logic. Sources: tensor_spec.hpp, buffer.hpp.
